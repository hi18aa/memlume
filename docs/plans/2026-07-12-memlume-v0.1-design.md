# Memlume v0.1 設計

## 目標

建立一個可公開使用的本地優先 Agent 共用記憶核心。v0.1 必須能以 CLI 與 MCP 提供最小必要 Context Pack，並以 SQLite 保存可追溯的事件與結構化記憶。

## 已核定範圍

- SQLite、Event Journal、Policy、Preference、Fact、Decision、FTS5、Context Resolver、CLI、Daemon、MCP Server。
- 支援手動寫入、搜尋與 Context Pack 查詢；資料只監聽 localhost。
- 首次公開版本為 `v0.1.0`，完成核心、三語 README 與發布檢查後推送 GitHub。

## 不在 v0.1 的項目

- 管理介面、向量資料庫、雲端同步、多租戶、模型微調、無限制自動改寫規則。

## 文件與識別

- 根目錄 `README.md` 使用英文，供 GitHub 首頁與國際使用者閱讀。
- `docs/README.zh-TW.md` 與 `docs/README.zh-CN.md` 提供繁體與簡體中文版；三份內容的功能、指令與限制必須一致。
- Logo 依 Mem（記憶）與 Lume（光）概念製作，最終提供 SVG、PNG、favicon 與 README 橫式版本。

## 依賴順序

1. 品牌規格與專案骨架。
2. contracts、Zod schema 與 SQLite migration。
3. Event Journal、記憶查詢與 FTS5。
4. Context Resolver、daemon、CLI 與 MCP。
5. 端對端範例、README、多語文件與發布檢查。

## 驗收條件

- MCP Client 可呼叫 `resolve_context` 並取得具來源的 Context Pack。
- 使用者明確規則可保存為結構化 Policy；事件與衍生記憶分開保存。
- CLI 的寫入、搜尋與 context resolve 均可於全新本機環境執行。
- 三語 README 均能完成安裝與最小範例。

## 發布前必要決策

- 選定開源授權條款。
- 選定最終 Logo。
- 建立 GitHub repository 與發布權限。
